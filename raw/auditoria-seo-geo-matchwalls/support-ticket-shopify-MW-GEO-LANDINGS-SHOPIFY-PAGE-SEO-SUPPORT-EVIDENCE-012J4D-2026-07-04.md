Store: `matchwalls`  
Domain: `https://www.matchwalls.com`

Base handle: `papel-pintado-espana`

Issue:

After briefly adding and then rolling back a native Shopify meta description, the storefront sometimes still returns the old value through `page_description`, even though Shopify Admin and API do not show an active SEO description field.

Expected behavior:

With `global.description_tag = null`, all requests to this URL should return either an empty `<meta name="description">` or a consistent body-derived fallback.

Actual behavior:

Bot-like user-agent requests intermittently return a stale 134-character value that does not match the current body summary or any visible Admin/API field.

Timeline:

- The native Shopify SEO description was added and then removed/rolled back on 2026-07-04.
- The stale value has been observed intermittently after rollback.
- Current Page `updatedAt` from Admin API: `2026-07-04T00:07:41Z`.
- This `updatedAt` timestamp appears to correspond to the rollback save that cleared the SEO description field. We have not intentionally made further content, title, body, handle, translation, market, or theme writes to this page after that rollback.
- We have not attempted a no-op save after preparing this ticket, because we want confirmation of the safest purge path before triggering any further write/update event.

`Papel pintado, murales y soluciones a medida para viviendas y proyectos profesionales en España. Pide muestras y personaliza tu mural.`
